HTML скелет — базова структура

Кожен HTML файл починається з одного і того самого шаблону. Це фундамент.
ℹ️
lang="uk" — атрибут мови на <html> важливий для пошукових систем і скринридерів. Кодування charset="UTF-8" підтримує українські символи.
HTML — index.html 
<!-- Перший рядок БУДЬ-ЯКОГО HTML файлу — тип документа -->
<!DOCTYPE html>
<html lang="uk">
<head>
  <!-- Кодування символів -->
  <meta charset="UTF-8">

  <!-- Viewport для адаптивного дизайну -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Підключення CSS файлу -->
  <link rel="stylesheet" type="text/css" href="/styles.css">

  <title>Заголовок - основна суть пропозиції</title>
</head>
<body>

  <!-- ... всі блоки сторінки йдуть тут ... -->

</body>
</html>
Тег / атрибут Навіщо потрібен Без нього
<!DOCTYPE html>Вмикає HTML5 режим браузераБраузер рендерить по-старому
charset="UTF-8"Кодування для кирилиці??? замість українських літер
viewport metaПравильне відображення на телефоніСайт зменшується до мурашиного розміру
link rel="stylesheet"Підключає зовнішній CSSСайт без жодних стилів

FAQ — часті запитання

FAQ toggle логіка

Скрипт розміщується перед закриваючим тегом </body>.
JS — index.html перед </body> 
<script>
  // Вибираємо всі елементи для кліку
  document.querySelectorAll('.faq-question').forEach(question => {

    // Для кожного питання — слухач на клік
    question.addEventListener('click', function() {

      // Знаходимо батьківський .faq-item
      const faqItem = this.closest('.faq-item');

      // Переключаємо клас .active
      faqItem.classList.toggle('active');
    });
  });
</script>
МетодЩо робить
querySelectorAll()Вибирає ВСІ елементи що збігаються з CSS селектором → NodeList
.forEach()Перебирає кожен елемент із NodeList
addEventListener('click', fn)Вішає обробник кліку на елемент
.closest(selector)Шукає найближчого батька що відповідає селектору
classList.toggle(class)Додає клас якщо немає, видаляє якщо є

Кращі практики розробки

✓
Mobile-first підхід
Пишіть базові стилі для мобільних, потім розширюйте для десктопу через @media (min-width: 768px).
✓
Семантичний HTML
Використовуйте <header>, <nav>, <main>, <section>, <footer> замість суцільних <div>.
✓
CSS Custom Properties
Зберігайте кольори, розміри і відступи в :root { --color-primary: #3b82f6; } для легкого редагування всього сайту через одне місце.

HTML Структура Документа

Базові елементи кожного HTML-документа — DOCTYPE, html, head, body та важливі мета-теги.
1
<!DOCTYPE html>
Декларація типу документа. Повинна бути першим рядком HTML файлу. Вказує браузеру, що це HTML5 документ.
2
<html lang="uk">
Кореневий елемент. Обгортає весь контент сторінки. Атрибут lang важливий для SEO та скринридерів.
3
<head>
Голова документа. Містить мета-інформацію, стилі, скрипти та інші невидимі елементи.
💡 Усі елементи в <head> невидимі для користувача, але впливають на сторінку.
4
<body>
Тіло документа. Містить весь видимий контент сторінки (текст, зображення, кнопки тощо).

Елементи Списків

Невпорядковані, впорядковані та списки описів — три типи списків в HTML.
•
<ul> — Невпорядкований
Список з маркерами (точками). Елементи не мають числового порядку.
1.
<ol> — Впорядкований
Список з нумерацією (1, 2, 3 або a, b, c). Для покрокових інструкцій.
≡
<dl> — Список описів
<dt> — термін, <dd> — опис. Для словників і глосаріїв.
HTML — Всі три типи списків 
<!-- Невпорядкований список -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Впорядкований список (нумерований) -->
<ol>
  <li>Крок перший</li>

</ol>

<!-- Список описів -->
<dl>
  <dt>HTML</dt>
  <dd>Мова розмітки гіпертексту</dd>
  <dt>CSS</dt>
  <dd>Таблиці каскадних стилів</dd>
</dl>

Як правильно підключити плагін

Три способи підключення — від найпростішого (CDN) до правильного для великих проєктів (npm).
HTML — Спосіб 1: CDN (для навчання і прототипів) 
<head>
  <!-- 1. CSS плагіна — у <head>, до своїх стилів -->
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/PLUGIN/dist/plugin.min.css">
  <link rel="stylesheet" href="styles.css">  <!-- свої стилі після -->
</head>
<body>
  <!-- ... контент ... -->

  <!-- 2. JS плагіна — перед </body> -->
  <script src="https://cdn.jsdelivr.net/npm/PLUGIN/dist/plugin.min.js"></script>
  <!-- 3. Свій JS після плагіна -->
  <script src="script.js"></script>
</body>
BASH + JS — Спосіб 2: npm (для реальних проєктів) 
# Встановлення через термінал
npm install swiper chart.js dayjs axios

// Імпорт у JavaScript файлі
import Swiper from 'swiper';
import { Chart } from 'chart.js';
import dayjs from 'dayjs';
import axios from 'axios';
🌿
Золоте правило: Не підключай плагіни про всяк випадок. Кожен плагін = зайві кілобайти які завантажує користувач. Підключай тільки те, що реально використовується на сторінці.
1
Перевір розмір плагіна
Для мобільних сайтів критично. Загальний JS не повинен перевищувати 200-300KB.
2
CSS завжди перед JS
Підключай CSS плагіна в <head>, JS перед </body>.
3
Свій код після плагіна
script.js завжди після всіх плагінів, інакше new Swiper() викине помилку.
4
Читай офіційну документацію
Кожен плагін має сайт з прикладами. Не копіюй з випадкових статей — вони можуть бути застарілими.
5
Перевіряй npm trends
npmtrends.com показує популярність бібліотек. Популярна = підтримується і безпечна.
