🗂️ Структура файлів проекту

Проект складається з трьох файлів в одній папці. Розділення HTML і CSS — обов'язкова практика.

📄
index.html
Вся HTML розмітка сторінки. Структура і контент.
🎨
styles.css
Всі CSS стилі. Зовнішній файл підключається через <link>.
📝
README.md
Документація проекту для розробника і команди.
bash
# Структура папки
Новая папка/
├── index.html   # ← головний HTML файл з розміткою сайту
├── styles.css   # ← CSS стилі для оформлення сайту
└── README.md    # ← документація

🏗️ HTML скелет — базова структура

Кожен HTML файл починається з одного і того самого шаблону. Це фундамент, без якого браузер не зрозуміє що відображати.

ℹ️

lang="uk" — атрибут мови на <html> важливий для пошукових систем і скринридерів. Кодування charset="UTF-8" підтримує українські символи.

HTML
<!-- Перший рядок БУДЬ-ЯКОГО HTML файлу — тип документа -->
<!DOCTYPE html>
<html lang="uk">
<head>
  <!-- Кодування символів: UTF-8 дозволяє правильно відображати українські літери -->
  <meta charset="UTF-8">

  <!-- Viewport для адаптивного дизайну: масштабує сторінку правильно на мобільних пристроях -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Підключення зовнішнього CSS файлу для всієї стилізації сайту -->
  <link rel="stylesheet" type="text/css" href="/styles.css">

  <!-- Заголовок у вкладці браузера -->
  <title>Заголовок - основна суть пропозиції</title>
</head>

<!--
  ОСНОВНИЙ КОНТЕЙНЕР СТОРІНКИ
  Усі явні елементи розміщуються всередину цього <body>
-->
<body>

  <!-- ... всі блоки сторінки йдуть тут ... -->

</body>
</html>
Тег / атрибутНавіщо потрібенБез нього
<!DOCTYPE html>Вмикає HTML5 режим браузераБраузер рендерить по-старому
charset="UTF-8"Кодування для кирилиці??? замість українських літер
viewport metaПравильне відображення на телефоніСайт зменшується до мурашиного розміру
link rel="stylesheet"Підключає зовнішній CSSСайт без жодних стилів

1 Блок: Header / Навігаційна панель

Забезпечує швидку навігацію по сайту. Видима на всіх сторінках (sticky/липка позиція). Містить логотип, основні посилання, кнопку логіну.

HTML — index.html
HTML
<!--
  СТРУКТУРА:
  .header — зовнішній контейнер з фіксованою позицією
  .nav — внутрішній контейнер з flex-layout для вирівнювання
  .logo — логотип зліва (flex: 0 0 auto)
  .nav-links — центральні посилання (flex: 1, gap: 30px)
  .btn-login — кнопка справа (flex: 0 0 auto)
-->
<header class="header">
  <nav class="nav">

    <!--
      ЛОГОТИП / БРЕНД
      • Розміщений зліва за допомогою flex
      • Має велику вагу шрифту (700) для видимості
      • Темний колір #1f2937 для контрасту
    -->
    <div class="logo">Логотип</div>

    <!--
      НАВІГАЦІЙНІ ПОСИЛАННЯ
      • Розташовані в центрі
      • flex: gap: 30px дає рівні проміжки
      • Посилання мають підтримку :hover для інтерактивності
      • id на секціях дозволяє перейти на них по кліку
    -->
    <div class="nav-links">
      <a href="#section1">Link 1</a>
      <a href="#section2">Link 2</a>
      <a href="#section3">Link 3</a>
      <a href="#section4">Link 4</a>
      <a href="#section5">Link 5</a>
    </div>

    <!--
      КНОПКА ВХОДУ
      • Розташована справа
      • Прозорий фон з тонкою межею
      • При наведенні (hover) змінює фон на сірий
      • Компактні розміри: padding 8px 16px
    -->
    <button class="btn-login">Вхід</button>

  </nav>
</header>
CSS — styles.css
CSS
/**
 * КОНТЕЙНЕР HEADER
 * position: sticky; top: 0 -> залишається закріплена в верхній частині при прокручуванні
 * z-index: 100 -> видна вище інших елементів (важливо для overlay)
 * background-color: #ffffff -> білий фон для контрасту
 * border-bottom: 1px solid #e5e7eb -> тонка лінія відокремлює від вмісту
 * padding: 12px 0 -> вертикальні вступи
 */
.header {
  background-color: #ffffff;
  border-bottom: 1px solid #e5e7eb;
  padding: 12px 0;
  position: sticky;
  top: 0;
  z-index: 100;
}

/**
 * NAV КОНТЕЙНЕР
 * max-width: 1200px -> обмежена ширина вмісту
 * margin: 0 auto -> центрування (0 верх/низ, auto боки)
 * display: flex -> активує flexbox layout
 * justify-content: space-between -> логотип ліворуч, посилання в центрі, кнопка праворуч
 * align-items: center -> вертикальне центрування елементів
 */
.nav {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 20px;
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/**
 * ЛОГОТИП / БРЕНДИНГ
 * font-weight: 700 -> жирний шрифт для виділення
 * color: #1f2937 -> темний колір для контрасту на білому фоні
 */
.logo {
  font-size: 20px;
  font-weight: 700;
  color: #1f2937;
}

/**
 * КОНТЕЙНЕР НАВІГАЦІЙНИХ ПОСИЛАНЬ
 * display: flex -> розташування елементів в ряд
 * gap: 30px -> відстань між кожним посиланням (30 пікселів)
 */
.nav-links {
  display: flex;
  gap: 30px;
}

/**
 * ОКРЕМЕ НАВІГАЦІЙНЕ ПОСИЛАННЯ
 * text-decoration: none -> видаляє підкреслення
 * color: #6b7280 -> світло-сірий колір
 * transition: color 0.3s -> плавна зміна кольору при наведенні
 * :hover -> color: #3b82f6 -> змінюється на яскравий синій колір
 */
.nav-links a {
  text-decoration: none;
  color: #6b7280;
  font-size: 14px;
  transition: color 0.3s;
}
.nav-links a:hover {
  color: #3b82f6;
}

/**
 * КНОПКА Входу (LOGIN)
 * background-color: transparent -> прозорий фон (мінімалістично)
 * border: 1px solid #d1d5db -> тонка світла межа
 * padding: 8px 16px -> компактний розмір
 * border-radius: 6px -> легка закругленість
 * transition: all 0.3s -> плавна анімація всіх змін
 * :hover -> background-color: #f3f4f6 -> світлий фон появляється
 */
.btn-login {
  background-color: transparent;
  border: 1px solid #d1d5db;
  color: #374151;
  padding: 8px 16px;
  border-radius: 6px;
  cursor: pointer;
  font-size: 14px;
  transition: all 0.3s;
}
.btn-login:hover {
  background-color: #f3f4f6;
}

2 Блок: Hero секція

Перше що бачить користувач. Ліва колонка — заголовок h1, підзаголовок, кнопка CTA. Права колонка — зображення або ілюстрація. Макет: CSS Grid, 2 рівні колони.

HTML — index.html
HTML
<section class="hero">

  <!--
    ЛІВА КОЛОНА — ВМІСТ
    • flex-direction: column для вертикального перекладання
    • gap: 24px для рівних відстаней між елементами
    • Розташовується над зображенням на мобільних пристроях
  -->
  <div class="hero-content">

    <!--
      ОСНОВНИЙ ЗАГОЛОВОК
      • font-size: 48px — експресивний, привертає увагу
      • line-height: 1.2 — щільна лінія для потужності
      • color: #1f2937 — темно-сірий, максимальний контраст
      • font-weight: 700 — bold для виділення
    -->
    <h1 class="hero-title">Заголовок — основна суть пропозиції</h1>

    <!--
      ПІДЗАГОЛОВОК / ОПИСАННЯ
      • font-size: 18px — дещо менше за основний для ієрархії
      • color: #6b7280 — світло-сірий для вторинної інформації
      • line-height: 1.6 — розширена для легкого читання
    -->
    <p class="hero-subtitle">Коротке пояснення пропозиції у
      одній-двох строках для заохочування дії користувача до дії</p>

    <!--
      КНОПКА ДІЇ (CTA - Call To Action)
      • .btn-primary — основна синя кнопка (#3b82f6)
      • padding: 12px 32px — достатній розмір для натискання
      • border-radius: 8px — м'які кути для сучасного вигляду
      • transition: all 0.3s — плавна анімація при :hover
      • При :hover: фон темніє, піднімається на 2px, додається тінь

    ПРАВА КОЛОНА — ЗОБРАЖЕННЯ
    • .hero-image — потужний контейнер (400px × 400px)
    • background-color: #f3f4f6 — світлий фон для контрасту
    • border-radius: 12px — закруглені кути для м'якості
    • display: flex + justify-content/align-items: center — центрування вмісту

    <div class="image-placeholder"></div>
  </div>

CSS

💡 Кращі практики

  • 01
    Один <h1> на сторінці — правило SEO. Всі інші: h2, h3 за ієрархією вниз.
  • 02
    box-sizing: border-box на * — перший рядок будь-якого CSS файлу. Padding входить у ширину.
  • 03
    position: sticky + z-index для header — щоб навігація залишалась видимою при прокручуванні.
  • 04
    transition: all 0.3s на hover-елементах — різка зміна без transition виглядає дешево.
  • 05
    max-width + margin: 0 auto — стандарт центрування контенту. Використовується в кожній секції.
  • 06
    id на секціях — id="section1" дозволяє навігації з href="#section1" скролити до потрібного місця.
  • 07
    flex-shrink: 0 на іконках — щоб іконка не стискалась коли тексту стає більше.
  • 08
    JS перед </body> — скрипти в кінці документа: HTML завантажується першим, потім JS знаходить елементи.
  • 09
    overflow: hidden на картках — щоб зображення всередині картки не виходило за межі border-radius.
  • 10
    cursor: pointer — на кожному клікабельному елементі. Без нього cursor стрілкою — заплутує користувача.

🚀 Як правильно підключити плагін

Три способи підключення — від найпростішого (CDN) до правильного для великих проєктів (npm).

Спосіб 1 — CDN (для навчання і прототипів)

HTML
<head>
  <!-- 1. CSS плагіна — у <head>, до своїх стилів -->
  <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/PLUGIN/dist/plugin.min.css">
  <link rel="stylesheet" href="styles.css">  <!-- свої стилі після -->
</head>
<body>
  <!-- ... контент ... -->

  <!-- 2. JS плагіна — перед </body> -->
  <script src="https://cdn.jsdelivr.net/npm/PLUGIN/dist/plugin.min.js"></script>
  <!-- 3. Свій JS після плагіна -->
  <script src="script.js"></script>
</body>

Спосіб 2 — npm (для реальних проєктів)

JS
# Встановлення через термінал
npm install swiper chart.js dayjs axios

# Імпорт у JavaScript файлі
import Swiper from 'swiper';
import { Chart } from 'chart.js';
import dayjs from 'dayjs';
import axios from 'axios';
🌿

Золоте правило: Не підключай плагіни про всяк випадок. Кожен плагін = зайві кілобайти які завантажує користувач. Підключай тільки те, що реально використовується на сторінці.

  • 01
    Перевір розмір — для мобільних сайтів критично. Загальний JS не повинен перевищувати 200-300KB.
  • 02
    CSS завжди перед JS — підключай CSS плагіна в <head>, JS перед </body>.
  • 03
    Свій код після плагіна — script.js завжди після всіх плагінів, інакше new Swiper() викине помилку.
  • 04
    Читай офіційну документацію — кожен плагін має сайт з прикладами. Не копіюй з випадкових статей.
  • 05
    Перевіряй npm trends — npmtrends.com показує популярність бібліотек. Популярна = підтримується і безпечна.
